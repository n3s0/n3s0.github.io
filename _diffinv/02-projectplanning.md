## Summary

This is the outline of my project DiffInv. An application that will
encapsulate a variety of IT related stuff. These things include the
following. I've always wanted an application that will do all of the
following for me. Without having multiple applications to turn to.

- IT Asset Management (Much like Snipe-IT)
  - Assets - Servers, workstations, networking gear, virtual machines,
    printers, etc.
  - Licensing for software
  - Users
  - Asset assignment to users
  - Asset assignment to assets
  - Consumable assignment to Assets
  - Accessories
  - Components
  - Consumables
- Infrastructure Resource Modeling (Much like netbox)
  - Sites
  - Wireless
  - IP Address Management
  - Equipment Racks
  - Servers
  - Network Gear
  - Connections
  - Virtualization
  - Data Circuits
  - VLAN Management
  - Contacts
  - Service and application matrix
  - Changelogs to infrastructure
  - Journaling or worklogs
  - Ability to build a logical topology with the data provided.
- Dashboard
  - Provides categorization
  - Provides links
  - Names
  - Some examples of URLs used:
    - Ticketing System
    - Firewall management portals
    - Admin portals
    - Monitoring application uis
- Configuration Management
- Tennant driven - The ability to serve this data for multiple
  customers.
- Multiple stub REST API for managing the application and making
  reporting available.


I don't know how I'm going to put all of this into one application so it
works well. All I know is that I've learned enough about these things a
little and would like to learn more. So, what better way to do this then
develop a fully functional application for it.

This project is going to be large. It will probably have a lot of moving
parts. So, with that in mind. There will be multiple sections associated
with it. Stay tuned. Each core concept will be planned and developed
individually and within the span that time allows.

I hope to see what this turns into.
